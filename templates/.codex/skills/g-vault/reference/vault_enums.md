# Vault Enums & Reference Tables

Canonical reference for vault note metadata values. Used by the `g-vault` skill.

## Frontmatter Fields

| Field | Required | Description |
|-------|----------|-------------|
| `date` | **yes** | When the note was created (YYYY-MM-DD) |
| `type` | **yes** | Content type — determines folder placement (see enum below) |
| `ingestion_type` | **yes** | How the note was ingested (see enum below) |
| `source` | **yes** | Origin URL or system identifier |
| `title` | **yes** | Human-readable title |
| `topics` | **yes** | Searchable topic labels (array, use multiple) |
| `project_id` | no | UUID if project-specific, `null` if global |
| `refresh_policy` | no | `none` \| `daily` \| `weekly` \| `monthly` \| `on_demand` |
| `refresh_after` | no | ISO date: when to re-fetch source (`null` = never) |
| `expires_after` | no | ISO date: when content is considered stale (`null` = never) |
| `source_volatility` | no | `low` \| `medium` \| `high` — how fast the source changes |
| `source_notes` | no | Array of wikilink paths (knowledge cards only) |

## Refresh & Expiration Policy

| Source Type | `refresh_policy` | `source_volatility` | `expires_after` |
|-------------|-------------------|---------------------|-----------------|
| YouTube video | `none` | `low` | `null` (permanent) |
| GitHub repo (active) | `weekly` | `high` | 30 days from refresh |
| Platform docs (Anthropic) | `weekly` | `high` | 14 days from refresh |
| Platform docs (stable) | `monthly` | `medium` | 60 days from refresh |
| Academic paper | `none` | `low` | `null` (permanent) |
| Web article | `on_demand` | `medium` | 180 days |
| Knowledge card | `on_demand` | `medium` | has own refresh logic |
| Math/science | `none` | `low` | `null` (permanent) |

When `refresh_after` date is reached, the `g-knowledge-refresh` skill will:
1. Re-fetch the source
2. Diff against existing content
3. Update the note or flag for human review
4. Reset `refresh_after` based on `refresh_policy`

When `expires_after` date is reached, the note is flagged as stale in `_index.yaml` and excluded from high-confidence search results.
